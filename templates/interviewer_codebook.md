# Interviewer Behavior Codebook (A4, Option 4)

> Code **every interviewer turn** in every pilot transcript with exactly one code — the dominant function of the turn. Openings/closings per the consent script are coded GUIDE. Report counts per transcript and per code in `VALIDATION.md`.

## GUIDE — On-guide question

A core question from the interview guide, asked at an appropriate moment, in faithful (not necessarily verbatim) form.
**Counter-example:** a guide question asked while the participant was mid-story on a related point → DRIFT or MISS depending on what was abandoned.

## PROBE — Appropriate probe

A follow-up that deepens what the participant just said: asking for an example, a clarification, a contrast ("how was that different from…"), or gentle silence-filling that returns the floor.
**Example:** P: "It changed how I plan my week." → I: "Walk me through last week — where did it show up first?"

## MISS — Missed probe opportunity

The participant offered a door — an emotion, a contradiction, a vague but loaded phrase ("it was fine, mostly") — and the interviewer moved to the next topic instead of walking through it.
**This is the hardest code to apply and the most informative one.** When in doubt between MISS and GUIDE, ask: would a trained interviewer have followed up here?

## LEAD — Leading question

A question that embeds its preferred answer, presupposes an unstated fact, or offers a ready-made interpretation.
**Examples:** "So that must have been frustrating?" · "Don't you think the tool saves you time?"
**Counter-example:** "How did that feel?" → PROBE.

## EMP — Unwarranted empathy-claim

The interviewer claims experience, feeling, or understanding it cannot have ("I know exactly how that feels," "I've been through that too") — distinct from neutral acknowledgment ("thank you, that's helpful"), which is coded with the turn's main function.

## DRIFT — Off-topic drift

A turn that serves neither the guide nor the participant's last answer: tangents, unsolicited advice, opinion-giving, answering its own question, or topic jumps that break the guide's logic without cause.

## Tie-breakers

- A turn with a leading *and* probing element → LEAD (errors dominate).
- Acknowledgment + next guide question in one turn → GUIDE.
- Can't decide MISS vs. DRIFT: MISS requires an identifiable door the participant opened; otherwise DRIFT.
- Code from the participant's perspective at that moment, not from what the transcript later reveals.
